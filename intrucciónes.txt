En general solo deben hacer esto:

Cada uno de los equipos debe desarrollar su microfrontend, usando una arquitectura de monolito
en cada servicio "Backend y Frontend juntos", pues no nos interesa hacer algo con arquitectura
desacoplada.

El proyecto es básicamente una app colaborativa de equipos, entonces uds le meterán imaginación
a su servicio/microfrontend. 

Para Auth es servicio de logueo "por lo que no tenemos BD puede funcionar de manera simbolica y ya"

Para Chat, basta con hacer un servicio de websockets entre usuarios o entre algun host 1 y host 2
"uds verán como lo hacen"

Para docs si no sabría que decirle, hagan una hoja que pueda modificarse entre usuarios y ya "Invente
Roman, invente" o si lo ven muy raro, no tiene que ser de docs, sino de otra cosa.

Para Tasks es solamente guardar tareas.

Una vez hecho eso, dentro de la carpeta de k8s deben:
- Crear archivo con el nombre de su servicio/microfrontend .yaml
- Modificar su estructura para ejecución, les tiro un ejemplo "Este viene de un proyecto pasado":
apiVersion: apps/v1
kind: Deployment
metadata:
  name: products
spec:
  replicas: 1
  selector:
    matchLabels: { app: products }
  template:
    metadata: { labels: { app: products } }
    spec:
      containers:
      - name: products
        image: yourrepo/products:latest
        ports: [{ containerPort: 80 }]
---
apiVersion: v1
kind: Service
metadata:
  name: products
spec:
  selector: { app: products }
  ports: [{ port: 80, targetPort: 80 }]
  type: ClusterIP


Una vez hecho todo lo anterior, ir a constructor.sh para ver el proceso de ejecución
